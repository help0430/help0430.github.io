認證與授權

認證(Authentication)是用來證明你是誰，
授權(Authorization)是用來決定你能幹嘛，
所以要先認證，知道你是誰之後才根據你的身分給予權力。

Claim、Identity、Principla

再來有幾個重要的名詞要搞清楚，

  1. Claim: 聲明，表示使用者的各種資料
  2. Identity: 身分文件，由多種 Claim 組成，表示一種身分
  3. Principal: 當事人，事件的主角，可以擁有許多身分

舉個例子:
當你要出國登機時，要先檢查護照，護照是一個 Identity，由你的護照號碼、名字、照片等 Claim 組成。
然後到達候機室，要檢查機票，機票是一個 Identity，由你的名字、飛機號碼、座位等 Claim 組成。
然後可以在根據你的VIP卡的等級決定你可以受到的服務，vip 卡是一個 Identity，由你的名字、VIP等級等 Claim 組成。
所以當你是為了坐飛機出國這件事的 Principal，你必須擁有三種身分文件。
而同時，你可以因為不同事件而擔任不同 Principal。

IIdentity

有了這些概念，就來看看 Asp.Net Core 是怎麼處理這些事情。
一個基本的 Identity 類別是 ClaimsIdentity，名字可以看出來是由多個 Claim 組成的 Identity。
重要的屬性和方法如下

  • AuthenticationType
    表示這個是哪種類型的認證，現實上的例子可能是護照，駕照之類的，Asp.Net 的世界可能就會是 Google、Cookie、Bearer。
  • IsAuthenticated
    是否已經通過認證，有些時候即使他有這個身分，但可能還沒通過認證，例如購物車的訪客，一樣有基本的身分，但若是要結帳，就要真正通過認證
  • Claims
    表示你身分的基本資料，在一個 Identity 內可以有很多個，例如姓名，電話等等。

如果我們建立一個 ClaimsIdentity 執行個體，並在建構時給定 AuthenticationType ，那麼 IsAuthenticated 就會是 true，這也是唯一設定的方式。

IPrincipal

Asp.Net Core 將使用者個認證相關資訊放在 HttpContext.User，
他是一個 ClaimsPrincipal 類別，擁有 IPrincipal 介面，
以下是他的簡化版本

  • Identity 是 Identities 內第一個 ClaimIdentity
  • Claims 是 Principal 層級的聲明
  • IsInRole 在 Asp.Net Core 的世界建議你用 Claim-base 的方式做認證與授權，所以只是為了向下相容。
